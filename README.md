# Evaluation of LiDAR-IMU SLAM Strategies

Comparative analysis of LiDAR-IMU SLAM strategies with LeGO-LOAM and LIO-SAM in a simulated ROS/Gazebo environment, using the Clearpath Husky A200 UGV and the Velodyne Puck VLP-16 sensor. It evaluates loosely coupled (LeGO-LOAM) and tightly coupled (LIO-SAM) approaches using point cloud and IMU data.

---

## Simulation Environment

The simulated environment used is the LaR (Robotics Laboratory) lab at UFBA, available in the [lar_gazebo](https://github.com/lar-deeufba/lar_gazebo) repository. The robot used is the Clearpath Husky A200 equipped with the Velodyne Puck VLP-16 sensor.

### Temporal Evolution of Errors

Temporal evolution of position error (meters) and yaw orientation error (degrees):

![Comparative Error Chart](images/grafico_comparativo_erros_unificados.png)

> **Note on LeGO-LOAM's Yaw error:** the absolute yaw error (~94°) occurs because LeGO-LOAM initializes in a local reference frame that is rotated relative to Gazebo's global reference frame. Relative position tracking is highly accurate — the yaw error does not reflect mapping degradation, but rather a reference-frame misalignment between the algorithm and the ground truth.

### Trajectory Accuracy

| Algorithm / Mode | Mean Pos Error (m) | Max Pos Error (m) | Mean Yaw Error (°) |
|:---|:---:|:---:|:---:|
| LeGO-LOAM (With IMU) | 0.0632 | 0.1549 | 94.8253 |
| LeGO-LOAM (Without IMU) | 0.0680 | 0.1620 | 94.7035 |
| LIO-SAM | 0.1470 | 0.2449 | 2.0562 |

## How to Use

### 1. Docker Setup

Clone this repository to a fixed location:

```bash
# Suggested path
/home/your_user/ros_workspace
```

Start the Docker container with ROS Melodic:

```bash
docker run -it \
  -v /home/your_user/ros_workspace:/root/catkin_ws \
  --name workspace_melodic \
  osrf/ros:melodic-desktop-full
```

Install the Python dependencies:

```bash
sudo apt update
sudo apt install python3-pip -y
```

---

### 2. Recording a New Bag (optional)

To record a new bag in the lar_gazebo simulated environment:

**Terminal 1** — bring up the Gazebo environment:
```bash
cd ~/lar_gazebo-noetic
./scripts/run_husky.sh
```

**Terminal 2** — start recording:
```bash
cd ~/lar_gazebo-noetic
./scripts/shell.sh
cd /ws/src/lar_gazebo/bags/
rosbag record /gazebo_ground_truth/odom /imu/data /tf /tf_static /velodyne_points \
  -O caminho_husky.bag
```

**Terminal 3** — control the Husky with teleop:
```bash
cd ~/lar_gazebo-noetic
./scripts/shell.sh
rosrun teleop_twist_keyboard teleop_twist_keyboard.py
```

Teleop controls:
- `u / i / o` — forward (left / straight / right)
- `j / k / l` — turn in place (left / stop / right)
- `m / , / .` — reverse (left / straight / right)

The bag used in this experiment is available for download on Google Drive:

📦 **[Download the bag — Google Drive](https://drive.google.com/drive/folders/1uE2QlnkfLwvhhe4cZrvPHQz-vqMgg5U5?usp=sharing)**

It is **1:52 min (112s)** long and contains the following topics:

| Topic | Msgs | Type |
|:---|:---:|:---|
| `/gazebo_ground_truth/odom` | 2802 | nav_msgs/Odometry |
| `/imu/data` | 5604 | sensor_msgs/Imu |
| `/tf` | 11208 | tf2_msgs/TFMessage |
| `/tf_static` | 1 | tf2_msgs/TFMessage |
| `/velodyne_points` | 1120 | sensor_msgs/PointCloud2 |

### 3. Running the Algorithms

Enable display access before starting:

```bash
xhost +local:docker
```

**Terminal 1** — start the algorithm and RViz:

```bash
docker start workspace_melodic
docker exec -it workspace_melodic bash

cd /root/catkin_ws
source devel/setup.bash
export LIBGL_ALWAYS_SOFTWARE=1

# For LeGO-LOAM:
roslaunch lego_loam run.launch

# For LIO-SAM:
roslaunch lio_sam run.launch
```

**Terminal 2** — play the bag:

```bash
docker exec -it workspace_melodic bash

cd /root/catkin_ws
source devel/setup.bash
cd /root/catkin_ws/src/bags
rosbag play caminho_husky.bag --clock
```

---

### 4. Collecting Results

Open a recording terminal **before** playing the bag.

#### LeGO-LOAM without IMU

In LeGO-LOAM's `utility.h` file, replace `imuTopic` with `/imu_false/data` so the algorithm ignores the IMU:

```bash
rosbag record -O comparacao_legoLOAM_semIMU.bag \
  /gazebo_ground_truth/odom /aft_mapped_to_init /tf /tf_static

rostopic echo -b comparacao_legoLOAM_semIMU.bag -p /aft_mapped_to_init \
  > lego_loam_sem_imu_68.csv

rostopic echo -b comparacao_legoLOAM_semIMU.bag -p /gazebo_ground_truth/odom \
  > ground_truth.csv
```

#### LeGO-LOAM with IMU

Restore `imuTopic` to `/imu/data` in `utility.h`:

```bash
rosbag record -O comparacao_legoLOAM_comIMU.bag \
  /gazebo_ground_truth/odom /aft_mapped_to_init /tf /tf_static

rostopic echo -b comparacao_legoLOAM_comIMU.bag -p /aft_mapped_to_init \
  > lego_loam_com_imu_68.csv
```

#### LIO-SAM

```bash
rosbag record -O comparacao_LIOSAM.bag \
  /gazebo_ground_truth/odom /lio_sam/mapping/odometry /tf /tf_static

rostopic echo -b comparacao_LIOSAM.bag -p /lio_sam/mapping/odometry \
  > lio_sam.csv
```

### 5. Generating the Analyses

With all CSVs generated, run the analysis scripts:

```bash
docker exec -it workspace_melodic bash
cd /root/catkin_ws/src/analises

# Trajectory comparison plot (Kabsch/SVD alignment)
python3 plotar_trajetorias.py

# Comparative error plot and table
python3 erros_comparativo.py
```
